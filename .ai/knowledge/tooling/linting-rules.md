# Linting Rules Reference

All rules are enforced by the pre-commit hook. ALL violations block the commit.

## C# Rules

| Rule | ID | What It Checks | Rationale |
|------|-----|----------------|-----------|
| Line limit | `line-limit` | Max 300 lines per .cs file | Forces small, focused classes |
| Namespace | `namespace` | Must contain `namespace R8EOX` | Prevents global namespace pollution |
| Legacy Input | `input-system` | Blocks `Input.GetKey/GetAxis/GetButton` | Project uses new Input System only |
| Public fields | `public-field` | Blocks public fields of common types | Use `[SerializeField] private` instead |
| One type per file | `one-type-per-file` | Max 1 distinct class/struct/enum/interface | Keeps files focused and findable |
| Top-down | `top-down` | Blocks cross-system `.Internal` imports | Enforces system boundary isolation |
| SendMessage | `banned-pattern` | Blocks `SendMessage`/`BroadcastMessage` | Untraceable string-based messaging |
| FindObjectOfType | `banned-pattern` | Blocks all `FindObject*` variants | Global reach bypasses parent hierarchy |
| Singleton | `singleton` | Blocks `.Instance` accessor | Bypasses dependency injection |

## Documentation Rules

| Rule | ID | What It Checks | Rationale |
|------|-----|----------------|-----------|
| Min content | `doc-min-content` | CLAUDE.md must have 3+ lines | Prevents placeholder-only docs |
| Heading | `doc-heading` | Must start with `#` heading | Ensures proper document structure |
| Purpose section | `doc-purpose` | Folder CLAUDE.md needs `## Purpose` | Documents folder intent |
| Frontmatter | `frontmatter` | Agent/skill .md needs YAML frontmatter | Required for Claude Code discovery |
| Knowledge min | `knowledge-min` | Knowledge docs need 5+ lines | Ensures substantive content |

## Files Skipped (Unity-generated)

- `Library/`, `Packages/`, `ProjectSettings/`, `Temp/`, `UserSettings/`
- `Assets/Plugins/Roslyn/`
- `*.Designer.cs`, `*AssemblyInfo.cs`
- Files with `// <auto-generated>` marker in first 5 lines

## Bypass

`git commit --no-verify` — emergency only, not recommended.

## IDE Enforcement

`.editorconfig` provides real-time feedback in Rider/VS for:
- PascalCase naming (classes, methods, properties)
- camelCase naming (private fields, locals, parameters)
- Interface `I` prefix
- Brace placement (Allman style)
- Suppresses CS0649 for `[SerializeField]` fields
